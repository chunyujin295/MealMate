## 二、微信小程序注册与认证

### 2.1 注册小程序账号

1. 访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角「立即注册」，选择「小程序」
3. 按提示填写邮箱、密码、激活账号
4. 选择主体类型：
   - **推荐**：企业或个体工商户（支持微信支付、分享等全部能力）
   - 个人主体限制较多（不支持支付、部分分享能力受限）

### 2.2 完成微信认证

1. 登录 [微信公众平台](https://mp.weixin.qq.com/) → 设置 → 基本设置
2. 点击「微信认证」→ 按流程提交企业资料
3. 认证费用：**300元/年**
4. 认证周期：通常 1-3 个工作日

### 2.3 获取 AppID

1. 登录公众平台 → 开发 → 开发管理 → 开发设置
2. 复制 **AppID（小程序ID）**，形如 `wx3e0f51ca7a41cc91`

### 2.4 配置小程序信息

1. 设置 → 基本设置 → 小程序名称：**吃了么**
2. 设置 → 基本设置 → 小程序头像：上传应用图标
3. 设置 → 基本设置 → 小程序介绍：填写简介
4. 设置 → 基本设置 → 服务类目：选择「工具 > 信息查询」或「生活服务 > 饮食健康」

## 三、开通微信云开发

### 3.1 开通步骤

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 左侧菜单 → 开发 → 云开发
3. 点击「开通」，同意协议
4. 选择**按量付费**模式（基础套餐免费额度足够前期使用）

### 3.2 创建云环境

1. 在云开发控制台点击「新建环境」
2. 环境名称：`mealmate`（可自定义）
3. 计费方式：按量付费
4. 创建完成后，复制**环境ID**，形如 `mealmate-0g6nf1o2c5e7a8b9`

## 四、百度AI菜品识别开通

### 4.1 注册并开通服务

1. 访问 [百度智能云控制台](https://console.bce.baidu.com/)
2. 注册账号并完成实名认证（个人即可）
3. 在产品列表中搜索「**图像识别**」
4. 找到「**菜品识别**」→ 点击「开通服务」
5. 选择按量计费（免费额度：1000次/月，超出后约 **0.004元/次**）

### 4.2 获取 API 密钥

1. 百度智能云控制台 → 右上角头像 → **Access Key管理**
2. 或直接访问：https://console.bce.baidu.com/iam/#/iam/accesslist
3. 点击「创建Access Key」
4. 记录以下两个值：
   - **API Key**（约24位字母数字）
   - **Secret Key**（约32位字母数字）

> **安全提醒**：Secret Key 仅显示一次，请立即保存。后续配置到云函数环境变量中，不会暴露到前端。

## 五、项目配置与启动

### 5.1 用微信开发者工具打开项目

1. 启动**微信开发者工具**
2. 选择「小程序」→「导入项目」
3. 目录选择：`d:\Code\MealMate`
4. AppID：填入你在步骤 2.3 获取的 AppID
5. 项目名称：`MealMate`
6. 点击「导入」

### 5.2 修改云环境ID

打开 `miniprogram/app.js`，找到第 7 行：

```javascript
wx.cloud.init({
  env: 'mealmate-0g6nf1o2c5e7a8b9',  // ← 改为你的云环境ID
  traceUser: true
});
```

将 `mealmate-0g6nf1o2c5e7a8b9` 替换为你在**步骤 3.2** 获取的环境ID。

### 5.3 修改 AppID

打开 `project.config.json`，找到：

```json
"appid": "wx3e0f51ca7a41cc91"
```

替换为你的小程序 AppID。

### 5.4 配置 AI 识别密钥

在微信开发者工具中：

1. 左侧文件树找到 `cloudfunctions/aiRecognition/`
2. 右键点击 → 「更多设置」→「上传环境变量」
3. 或在微信开发者工具顶部菜单：
   - 云开发 → 云函数 → 找到 `aiRecognition`
   - 点击「配置」→「环境变量」
4. 添加以下两个环境变量：

| 变量名 | 值 |
|--------|-----|
| `BAIDU_API_KEY` | 百度 API Key（步骤4.2获取） |
| `BAIDU_SECRET_KEY` | 百度 Secret Key（步骤4.2获取） |

## 六、云函数部署

### 6.1 部署所有云函数

在微信开发者工具中，对每个云函数执行以下操作：

1. 在左侧文件树展开 `cloudfunctions/` 目录
2. 右键点击云函数文件夹（如 `login`）
3. 选择「**上传并部署：云端安装依赖**」

需要部署的云函数列表：

| 云函数 | 用途 | 部署顺序 |
|--------|------|----------|
| `login` | 微信登录与用户初始化 | 1 |
| `users` | 用户信息管理 | 2 |
| `records` | 饮食记录 CRUD + 统计 | 3 |
| `aiRecognition` | AI 菜品识别 | 4 |
| `groups` | 群组管理 | 5 |
| `initDB` | 数据库集合初始化 | 6 |

### 6.2 验证部署结果

1. 微信开发者工具 → 顶部「云开发」按钮
2. 进入云开发控制台
3. 左侧菜单 → 云函数
4. 确认 6 个云函数全部显示且状态为「部署成功」

## 七、数据库初始化

### 7.1 创建数据库集合

**方法一：通过云函数自动创建（推荐）**

1. 微信开发者工具 → 顶部「云开发」→ 云函数
2. 找到 `initDB` 云函数 → 点击「测试」
3. 直接点击「发起调用」
4. 返回结果应显示 5 个集合创建成功

**方法二：手动创建**

1. 微信开发者工具 → 顶部「云开发」→ 数据库
2. 点击「添加集合」，依次创建以下 5 个集合：

| 集合名称 | 用途 |
|----------|------|
| `users` | 用户信息 |
| `food_records` | 饮食记录 |
| `groups` | 群组信息 |
| `group_members` | 群成员关系 |
| `group_interact` | 群组互动（评论/点赞） |

### 7.2 配置数据库安全规则

1. 云开发控制台 → 数据库 → 选择集合
2. 点击「权限设置」标签
3. 参考 `cloudfunctions/database_security_rules.json` 文件内容配置每个集合的权限

**各集合权限规则：**

**users 集合：**
```
读：auth.openid == doc._openid
写：auth.openid == doc._openid
```

**food_records 集合：**
```
读：auth.openid == doc._openid || doc.visibility != 'private'
写：auth.openid == doc._openid
```

**groups 集合：**
```
读：所有用户可读
写：auth.openid == doc.ownerId
```

**group_members 集合：**
```
读：auth.openid == doc._openid
写：auth.openid == doc._openid
```

**group_interact 集合：**
```
读：所有用户可读
写：auth.openid == doc._openid
```

### 7.3 创建数据库索引（优化查询性能）

在云开发控制台 → 数据库 → `food_records` 集合 → 索引管理：

| 索引字段 | 索引类型 | 说明 |
|----------|----------|------|
| `_openid` | 普通索引 | 用户数据查询 |
| `_openid + date` | 联合索引 | 按用户+日期查询记录 |
| `_openid + isDeleted` | 联合索引 | 过滤已删除记录 |
| `date` | 普通索引 | 按日期筛选 |

## 八、本地调试与测试

### 8.1 基础功能测试流程

按以下顺序验证各功能模块：

**1. 登录功能**
- 打开小程序 → 首次自动登录
- 检查云开发 → 数据库 → `users` 集合，是否新增了用户记录

**2. 饮食记录**
- 首页点击「+」或「添加」→ 选择餐次类型
- 输入文字内容（如「米饭+番茄炒蛋」）
- 上传 1-2 张图片
- 点击保存 → 返回首页确认记录已展示

**3. AI 识别**
- 新增记录时，上传菜品照片
- 点击「AI 识别食物」按钮
- 等待识别结果回填至营养成分
- 可手动修改识别结果

**4. 历史查看**
- 首页点击日期切换 → 可查看历史日期记录
- 切换到「历史记录」页面 → 日历视图/列表视图切换

**5. 营养统计**
- 个人中心 → 营养统计
- 切换今日/本周/本月查看数据汇总

**6. 群组功能**
- 群组页面 → 创建群组
- 点击邀请好友 → 生成分享卡片
- 群组动态流中可查看成员公开记录

**7. 个人设置**
- 我的页面 → 修改昵称/头像
- 设置默认隐私选项

### 8.2 真机调试

1. 微信开发者工具 → 顶部「真机调试」
2. 使用微信扫码
3. 在手机上操作各功能模块
4. 检查图片上传、分享卡片等功能在真机上的表现

### 8.3 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 云函数调用失败 | 云函数未部署 | 重新上传部署云函数 |
| 云函数调用失败 | 环境ID不正确 | 检查 app.js 中的 env 值 |
| 数据库读写失败 | 集合未创建 | 运行 initDB 云函数 |
| 数据库读写失败 | 权限规则错误 | 检查安全规则配置 |
| AI识别失败 | 百度密钥未配置 | 配置云函数环境变量 |
| AI识别失败 | API额度用完 | 检查百度控制台用量 |
| 图片上传失败 | 云存储未开通 | 在云开发控制台开通 |
| 分享功能异常 | 未通过审核 | 需提交审核后才可正式使用 |
| 真机无法打开 | AppID错误 | 检查 project.config.json |

## 九、发布上线

### 9.1 提交审核前检查清单

- [ ] 所有页面 UI 正常显示，无白屏或错乱
- [ ] 登录、注册流程正常
- [ ] 饮食记录 CRUD 功能正常
- [ ] 图片上传、展示正常
- [ ] AI 识别功能正常
- [ ] 分享卡片生成正常
- [ ] 群组功能正常
- [ ] 小程序名称、图标已设置
- [ ] 服务类目已选择正确
- [ ] 用户隐私保护指引已设置
- [ ] 无测试数据残留
- [ ] 小程序包体积不超过 2MB（主包）+ 20MB（分包）

### 9.2 上传代码

1. 微信开发者工具 → 右上角「上传」
2. 版本号：`1.0.0`
3. 项目备注：`首次上线：基础饮食记录 + AI识别 + 群组社交功能`
4. 点击「上传」

### 9.3 提交审核

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 管理 → 版本管理
3. 在「开发版本」中找到刚上传的版本
4. 点击「提交审核」
5. 填写审核信息：
   - 功能页面截图（首页、记录页、营养统计页、群组页）
   - 测试账号（如有需要登录才能使用的功能）
   - 功能说明：
     ```
     本小程序为个人饮食健康管理工具，主要功能包括：
     1. 三餐饮食记录与拍照
     2. AI食物识别与营养分析
     3. 好友群组饮食打卡
     4. 营养统计与历史查看
     不涉及UGC社交发布功能，群组为私密好友圈。
     ```
6. 提交审核

### 9.4 审核周期

- 通常 **1-7 个工作日**
- 首次提交审核时间可能较长
- 如被驳回，根据驳回原因修改后重新提交

### 9.5 发布上线

审核通过后：

1. 公众平台 → 版本管理 → 审核版本
2. 点击「提交发布」
3. 选择**全量发布**
4. 发布后约 **5-30 分钟**全网生效
